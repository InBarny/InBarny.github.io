25
 1. Решает ли VR приложение обозначенную проблему (0-3 баллов). 1. Разработанное приложение в формате *.apk, *.exe или *.vwp (кнопка “Экспорт проекта целиком”). 1. Проект должен иметь осмысленный сценарий по собственному техническому заданию обучающихся (например: физический эксперимент, обучающий тренажер, логическая головоломка, интерактивный квест, сюжетная история и др.)
26
 2. Техническая реализация VR приложения (отсутствие ошибок в логике) (0-3 баллов). 2. Видео с демонстрацией работы приложения в Desktop-режиме длительностью до 3х минут в формате *.mp4. 2. В проекте должно присутствовать минимум две сцены и реализованы переходы между ними.
27
 3. Художественная реализация VR приложения (визуальный дизайн приложения) (0-3 баллов). 3. Презентацию* с описанием цели, задач, этапов работы над проектом, краткого сценария работы приложения, перспектив дальнейшего использовании приложения, трудностей, с которыми столкнулись при разработке. 3. В проекте должна использоваться минимум одна переменная, встроенная в логику (например, для подсчета баллов).
28
 4. Удобство интерфейса программного VR приложения для пользователя (UX/UI) (0-3 баллов). *Рекомендации по содержательному наполнению слайдов. Стиль оформления слайдов презентации – фирменный стиль КМЭПТ: 4. В проекте должен быть реализован минимум один список, состоящий минимум из трех элементов. Список должен быть встроен в общую логику сценария (например, для реализации движения объекта по маршруту).

 5. Разработанность сценария (сюжетной линии) VR приложения (0-3 баллов). [Слайд 1] Титульный слайд 5. В проекте должна быть реализована механика ограничения времени на выполнение заданий сценария.

 6. Оригинальность решения (0-3 баллов); Название проекта, команды, короткое представление 6. В проекте должна использоваться минимум одна зона, встроенная в логику сценария (например, для телепортации игрока на другую сцену).

 7. Качество презентации VR приложения: раскрыты ли цели, задачи, этапы работы над проектом, перспективы дальнейшего использования приложения (0-3 баллов) 7. В проекте должны быть использованы User Interface элементы (UI), которые должны сообщать о ключевых событиях (как минимум, старт и завершение сценария).
